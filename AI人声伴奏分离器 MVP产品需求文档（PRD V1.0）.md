# 2\. 用户核心流程（User Flow）

**完整链路**：用户打开本地网页 → 选择/拖拽本地音频文件 → 触发分离任务 → 前端展示加载状态 → 后端串行执行AI分离 → 生成本地音频文件 → 前端渲染双轨播放器 → 用户试听/下载音频

**分步详细流程**：

1. 用户本地打开 index\.html 前端页面，进入工具主界面；

2. 通过点击上传或拖拽方式，选取本地MP3/WAV音频文件；

3. 文件校验通过后，「开始分离」按钮点亮可点击；

4. 用户点击开始分离，前端展示Loading状态，禁止重复操作；

5. 后端接收音频文件，存入临时目录，开启任务互斥锁，进入单线程排队；

6. 后端调用AI模型执行人声、伴奏分离算法；

7. 算法处理完成，生成人声、伴奏双轨音频文件，迁移至静态资源目录，释放任务锁；

8. 前端接收返回路径，渲染双轨音频播放模块；

9. 用户可在线拖拽进度条试听，点击下载按钮将音频保存至本地。

# 4\. 非功能性需求

## 4\.1 性能与并发约束

- **并发机制**：本地单线程串行队列，全局互斥锁控制，同一时间仅执行一个分离任务，新任务自动等待上一任务完成

- **处理耗时标准**（以4分钟标准MP3音频为基准）：
        

    - CPU环境：90s \- 180s

    - GPU（CUDA加速）环境：15s \- 30s

- **资源限制**：单音频文件上限15MB，规避本地内存溢出、算力过载问题

## 4\.2 本地存储策略

- 目录分工：`temp/` 存储用户上传原始临时音频；`static/` 存储分离完成的结果音频

- 自动清理机制：后端服务每次启动时，自动清空 temp、static 目录下所有历史文件，避免本地磁盘冗余堆积

- 前端：兼容Chrome、Edge主流桌面浏览器

- 后端：支持Windows、macOS、Linux全平台本地部署运行

# 7\. 风险与约束说明

- **性能风险**：纯CPU环境处理速度较慢，属于正常现象，需在页面隐性提示用户耐心等待；

- **算法效果约束**：重度混响、多层编曲、特殊音效的复杂歌曲，分离精度会小幅下降，为模型正常上限，MVP不做极致效果优化；
